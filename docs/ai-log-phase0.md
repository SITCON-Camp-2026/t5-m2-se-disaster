# Phase 0 AI 使用紀錄

## 2026-07-09 快速衝刺

### 小組要求

用 coding agent 在 50 分鐘內做出一個會暴露資料品質問題的整理工作台。

### Agent 做了什麼

1. **分析 12 筆原始資訊**
   - 識別出 6 筆資訊提供具體判斷（M-003, M-007, M-009, M-010, M-011, M-012）
   - 其他 6 筆保留安全預設

2. **實作功能**
   - 在 App 層級加上 `drafts` 狀態管理
   - 在 Phase0JudgementCard 加編輯功能支持人類備註
   - 更新工作台連接編輯流程

3. **提供 6 筆資訊的具體判斷**

   | 資訊  | 類型     | 信心   | 重點風險                         |
   | ----- | -------- | ------ | -------------------------------- |
   | M-003 | 地點狀態 | 中     | 需與最新資訊交叉確認             |
   | M-007 | 人員指派 | 低     | 操作者非當事人，名單已過期       |
   | M-009 | 公告候選 | 中     | 與官方公告不同步                 |
   | M-010 | 地點狀態 | **高** | 最具體的資訊，但時效性短         |
   | M-011 | 求助候選 | 中     | 操作者非當事人，隱私待確認       |
   | M-012 | 求助候選 | 低     | 操作者非當事人，位置不明，涉醫療 |

### 人類的判斷和取捨

**採用了**

- ✓ 全數 6 筆的具體判斷
- ✓ 其他 6 筆的安全預設策略（保守，不亂補）

**修正了**

- M-007：Agent 初期傾向相信社群貼文 → 人類改為「操作者不是工班本人」標記為低信心
- M-011/M-012：Agent 初期想直接建立任務 → 人類改為「隱私優先、當事人確認」

### Agent 亂補或不能相信的地方

1. **sourceType 標籤 ≠ 內容可信度**
   - M-005 marked 「官方公告」但實是「截圖的不確定內容」
   - Agent 容易被標籤迷惑

2. **時間戳 ≠ 永遠成立的現場狀況**
   - M-010 14:35 的物資數據會在 16:30 改變
   - Agent 容易信任最新時間戳就當成不變事實

3. **相同物資的多個回報需謹慎比較**
   - M-002「不知道下午還有沒有」vs M-004「很多」
   - Agent 容易只看時間新舊，忽略來源信度差異

4. **代理人表達 ≠ 當事人需求**
   - M-011 志工轉述 vs 長者原意可能有落差
   - M-012 家屬推測 vs 親友實況可能誤判
   - Agent 容易忽視代理人問題

5. **缺失資訊 + 直觀判斷 = 容易誤補**
   - M-001 位置不明 → Agent 容易想「現場問路就行」
   - M-008 原因不明 → Agent 容易推測「可能道路危險」
   - M-012 位置缺失 → Agent 容易想「家屬應該能找到」
