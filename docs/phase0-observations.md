# Phase 0 Observations

## 我們做出了什麼？

- 整理工作台，支持查看 12 筆原始資訊
- 為 6 筆資訊 (M-003, M-007, M-009, M-010, M-011, M-012) 提供具體的候選判斷
- 顯示每筆資訊的來源、查核狀態、判斷依據和卡住的地方
- 支持人類審查備註編輯功能

## 哪些資料讓 agent 很容易整理？

- **M-010**：值守志工的現場更新，具體數據（雨鞋 12 雙、尺寸 26-28）、物資狀態、轉運指示、更新計畫。品質最高。
- **M-009**：現場志工 14:20 的回報，集合點規則明確、限制條件清楚、有物理位置（站前遮雨棚）。
- **M-003**：現場回報說資源狀態改變（不缺鏟子改需水電），承認可能過時，利於後續追蹤。

## 哪些資料讓 agent 亂補或誤判？

- **M-001**：容易把「需要十幾個人」直接當成任務派人，但位置「只有老雜貨店後面」無法派出，容易指派失敗。
- **M-002 vs M-004**：相同物資（雨鞋）但相互矛盾，一個說「不知道下午還有沒有」，一個說「很多」。Agent 容易挑一個當真。
- **M-005**：來源標示「官方公告」但內容是「截圖寫『中午前道路封閉』」，缺時間和來源。Agent 容易相信「官方」標籤。
- **M-006**：兩個志工意見衝突（可以 vs 淹水），Agent 容易挑一個當判斷依據。
- **M-008**：「A 區先不要再派人」沒說理由，Agent 容易亂補原因（人太多？道路危險？任務完成？）。

## 哪些資訊不能直接變成任務？

- **M-001**：位置不明（「老雜貨店後面」）
- **M-003**：原文提到鏟子與水電，但內容是「已不缺鏟子、比較需要水電」，不能直接變成準備鏟子或派水電任務。
- **M-010**：雨鞋數量看起來具體，但仍是當下盤點；不能保證後續仍有 12 雙，也不能直接當作可領取承諾。
- **M-011**：需要搬動大型家具，但位置不完整且尚未確認長者是否同意公開完整地址。
- **M-005**：不是明確的現場狀態，是截圖的不確定公告
- **M-008**：沒說是什麼狀況，無法建立清楚的任務
- **M-012**：家屬代表且無確切位置，涉及醫療判斷

## 工具與物資線索要怎麼看？

- **鏟子**：M-003 只說老街口「已不缺鏟子」，不能寫成需要準備鏟子；M-001 的清泥需求也沒有明確列工具。
- **雨鞋**：M-002、M-004、M-010 都提到雨鞋，但數量、時間與來源互相不一致或可能過期，需要現場盤點。
- **水電**：M-003、M-007、M-010 都提到水電，但有過期名單、登記地點和需求內容不明等問題，不能直接派工。
- **清泥 / 清淤工具**：M-001、M-009 只提供工作類型或集合限制，還缺地點、安全與工具種類確認。
- **搬運大型家具工具**：M-011 需要先確認當事人同意、位置和現場安全。
- **藥品協助**：M-012 涉及醫療判斷，只能標成需要人工確認。

## 出發前小卡要確認什麼？

- **食物**：目前原始資訊沒有食物供應或食物需求，不能推測現場有吃的。
- **飲用水**：M-010 說「飲用水暫時不缺」，但這只是當下線索，仍要確認目前是否還足夠、能不能領取或是否不要再送。
- **休息或睡覺地點**：目前原始資訊沒有住宿、過夜、休息或公園可停留資訊，不能把活動中心或集合點推測成可以睡的地方。
- **道路與路況**：M-005 提到道路封閉截圖，M-006 提到淹水，M-009 提到入口公告，M-008 提到不要再派人；這些都不能直接合併成可通行判斷。
- **車輛大小**：目前原始資訊沒有小卡車、大卡車或貨車限制，不能直接決定車型。

## 哪些資訊的操作者不是當事人？

- **M-007**：社群使用者說工班可支援，但操作者不是工班本人
- **M-011**：志工代長者轉述，操作者不是長者本人，且隱私待確認
- **M-012**：外地家屬來電，操作者距離遠，無法確認親友狀況

## 我們剛剛偷偷假設了什麼？

- 假設志工回報的時間戳就等於現場狀況的最新時間（實際上 M-010 的 14:35 現在已經過時）
- 假設「官方公告」的 sourceType 標籤就能保證內容可信（M-005 其實是截圖的不確定內容）
- 假設相同物資的不同回報能用「較晚的更新時間」判斷，卻忽略了來源可信度（M-002 vs M-004）
- 假設現場志工的判斷就等於當事人的需求（M-011, M-012 的代理人問題）

## 如果要讓這個工具變可靠，下一步需要什麼？

- **確認機制**：在分配任務前必須與當事人直接確認，不能只靠代理人
- **位置驗證**：位置資訊必須到「完整地址」或「GPS 座標」，不能只有相對方位
- **衝突解決**：同類資訊出現矛盾時（M-002 vs M-004），需要明確的優先級或最新驗證
- **隱私保護**：涉及個人資訊時必須明確記錄當事人同意
- **時間追蹤**：物資或現場狀況需要定時更新，不能信任單次回報
- **來源追蹤**：區分「官方發佈」、「志工轉述」、「社群貼文」、「家屬說法」的可信度差異
- **醫療判斷**：涉及藥品、醫療需求必須轉介醫療人員，不能由志工直接派任

## 我們需要更清楚資料格式的地方

- 出發前資訊需要分開記錄：食物、飲用水、休息或睡覺地點、道路與路況、車輛大小，不應混在任務描述裡。
- 每個項目都需要有「原文線索」「缺少資訊」「需要人工確認問題」，避免 agent 把缺資料補成答案。

## 我們需要人工確認的地方

- 原文提到的工具或物資，究竟是「需要準備」、「不缺」、「可領取」還是「請不要送」，必須分開確認。
- 志工出發前是否有食物、飲用水、可休息或睡覺地點、最新路況，以及適合小卡車或大卡車，都不能由目前原始資訊直接判斷。
